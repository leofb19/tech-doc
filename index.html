Getting Started

HTML stands for HyperText Markup Language. It is the standard language used to create documents designed for web browser usage and display. It includes useful formats and presets that allow easy modification and creation of well-written and formatted documents. These include something as simple as dotted lists, to embedded videos within a website. HTML is a versatile and useful language that serves as the foundation for web development and maintenance.

There have been several versions of HTML, starting with basic and evolving into a more complex and versatile language. The currently accepted and used version of HTML globally is HTML5. It included upgrades to and compatibility for:

  • Audio and video support
  • Canvas elements
  • Form enhancements and customization

HTML is used on every website as the barebone format and markup. It is then styled with CSS and made interactive and useful with JavaScript. Now that you have learnt the basic information surrounding HTML, let's begin with the more detailed information.

Basics

HTML uses elements/tags and attributes to tell the browser what to do with a certain lines of code, and how to display text and visuals. There are many of them, but for basic text display and formatting, the following are most widely used:

< h1 >< /h1 > < h2 >< /h2 > < h3 >< /h3 >

are used to create different headings. There are a total 6 types: h1 through h6. 1 is the largest, and 6 the smallest. They are helpful to make the beginning of a section of your document more obvious and important.

< p >< /p >

is used to create a paragraph, the most basic form of text. It automatically creates a new line everytime the tag is used for text.

< span >< /span >

is used to either create basic text that can then have format applied to it, or to wrap a sentence or few words within another element to apply format. Unlike the p element, it does not create a new line at the end.

< strong >< /strong >

makes text bold.

< em >< /em >

makes text italic.

< sup >< /sup > and < sub >< /sub >

make text super- and sub- scripted respectively.

< del >< /del >

makes text have a strike through.

Lists

The next useful element of HTML is the list element. As its name suggests, this element is used to create a list. However, there are two different types of lists that share similar syntax.

< ul > < li >< /li > < li >< /li > < /ul >

This syntax creates what is known as an unordered list. This list has no order, meaning it will display with bullet points by default.

Images

Images are used throughout websites for several reasons. They:

  • Provide character and style to a website
  • Convey visual data or examples
  • Add color and depth to certain sections to draw attention

Adding images to your document is extremely easy with HTML.

< img src="" alt="" >

This is all you need to be able to add an image to your website. The src="" attribute is where you input the source of the image. This needs to be a valid source, and must be accessible on some online platform so that all users can load it. The alt="" attribute is where you can enter a text alternative to be displayed if a user cannot load the image. It is considered a best practice to include an alt for every image, and make it a detailed description of the image for users who cannot see it.

< img src="" alt="" width="" height="">

Including the width="" and height="" attributes allows you to edit the size of the image and make it smaller or bigger.

Notice how the image tag doesn't have a second closing one? That is because the image tag is what we call a self-closing tag.

Forms

Forms in HTML include arguably the most elements and posibilities. Each element within a form can have specific rules and conditions, making them perfect for things like sign-up pages and, well, forms. Unironically.

The basic syntax for a form element is as follows:

< input type="" >

The input tag is also self-closing, meaning that this is all you need to create a form element. As the name suggest, this element allows a user to input something. The type="" selector is where you can determine what type of information you wish the user to enter. HTML 5 uses this information to then also validate the input to see if it matches what you want. There are several types which each do exactly what you'd think, as well as providing neat little features. They include:

  • text (creates a simple box for a user to enter text e.g. a name)
  • password (hides the user's input by replacing each character with a ·)
  • email (validates whether the user includes an @ symbol as well as a valid email domain)
  • age (can be given an age limit to only allow a user within a certain age group/range to submit the information)
  • checkbox (creates a checkbox that can be checked and unchecked)
  • radio

Everything is quite straightforward other than radio. A radio is similar to a checkbox, except that it cannot be unchecked once checked. Its main purpose is to be related with a name="" attritbute. When linked together, it creates a list of options that a user must choose from. To do this, put them in a list and tie them together as follows:

< input type="radio" name="radio" >

There are a few other useful attributes that you can add to an input type to make the HTML have stronger validation behavior and cleaner, more user-friendly format. These are:

< input type="" name="" value="" placeholder="" required checked >

Required creates a rule that the input must have a valid entry otherwise the form will not be submittable. The checked attribute is useful for checkboxes and radio buttons. It does as the name suggests, and applies a "checked state" to the input, making it mandatory for the user to select an option, otherwise the form will submit with the default checked option. The placeholder attribute allows you to enter a piece of text that will display if the user hasn't entered anything yet. This is helpful to provide information on what type of information should be entered, and can even include examples.

Buttons, textareas, and drop-down menus are also useful elements of a form. Each has it's own type, and have their own default behavior.

< input type="submit" value="" >

The submit input type is a button with pre-programmed behavior to submit a form, and displays the word "Submit" in it by default. Giving it a custom value attribute will change the default word that is displayed, and it can be changed to anything.

< input type="button" value="" >

The button input type creates an empty button with no pre-programmed behavior. It is useful to create a fully custom button with JavaScript, but in a normal HTML/CSS sense, it is dificult to use and a submit type input is more useful.

< textarea placeholder="" >< /textarea >

The textarea is not necessarily an input type, and has it's own syntax, needing both opening and closing tags. It is useful if you want the user to input a larger description in a customer support scenario, for example.

< select required > < option disabled >< /option > < option >< /option > < /select >

The select element also has it's own syntax, and functions just like an HTML list, replacing list-items with options. Adding the disables attribute to an option will make it unselectable, and is useful to act as a placeholder.

Tables

When displaying data, there are a two main methods that are clearly the best:

  • Visual (images or videos)
  • Tables

We already covered adding an image to your website, and adding videos will be covered next. But tables have, since their creation, been the most common and applicable method to display and organize collected data of any kind. Therefore, knowing how to create one is pivotal if you wish to create a clean and dense display of your data. The syntax takes ideas from an HTMl list, but has a few added tags.

< table > < tr > < th >< /th > < th >< /th > < /tr > < tr > < td >< /td > < td >< /td > < /tr > < /table >

This may look confusing and slightly menacing, but that's only because indents would make it more concise. The table element has both an opening and closing tag. The tr or table row tag creates a new row in the table. These are the horizontal components, and determine the height of a table. The th or table header tag creates a table header similar to a normal header. It defaults to bold text and is center aligned. The td or table data tag add table data, which is the actual text or whatever data you wish to represent. It defaults to normal text and is left aligned. Adding a table header or table data element within a row will increase the number of columns.

Similar to images, tables can have custom width and height for their cells, but not for the entire table. That is determined based on the number of rows and columns, and the total height and width of each cell combined.

Tables can also have a custom border by adding the border attribute within the opening tag. This attribute can contain a number of

< table border="2px"> < /table >

Something that is important to remember is:

  • The greatest height of any single cell within an entire row will determine the height of all cells within that row
  • The greatest width of any single cell withing an entire column will determine the width of all cells withing that column
  • Each row and column can have their own height and width respectively, but those values are determined by the above conditions
Links and iFrames

The final important part of basic HTML elements includes:

  • Links and
  • iFrames

A link is very well known. It's is how you got to this website.

< a href="" target="" >< /a >

The basic syntax includes an opening and closing tag, where the a represents the word anchor. A link in a website can either link to another website, or a another element/section within the same document/website. That is how the navigation links on the left are created, although that is an oversimplification.

The target="" attribute is what tells the anchor where and how to open. The default behavior of a link is to open and replace the current website. This is normal for all Google links for example. When you click on one, the same page you were viewing is now changed to the website link you clicked on. However, by changing the value of the attribute to _blank, the website will open in a new tab/window, keeping the original page you were on open. It looks like this:

< a href="" target="_blank" >< /a >

The space between the tags is how you make the website display. If left blank, there will be nothing to show. By adding text between the opening and closing tags, the link will now be "hyperlinked" to the text. This creates default formatting of a blue color and underline, and when hovered over or clicked, changes color. To see this in action, look at the references at the bottom of the page!

The final element we will discuss is the iFrame element.

< iframe src="" frameborder="0" >< /iframe >

Similar to an image, an iFrame needs a source. This is normally a video from somewhere like YouTube. There are opening and closing tags, and entering something inside them isn't necessary as it doesn't really do anything. When adding an iFrame from YouTube to your HTML, you simply need to copy the prepared code that is already given, making it much easier to use.
